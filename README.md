# 订单明细处理工具

一个基于浏览器的Excel数据处理工具，用于自动化处理订单数据并生成标准化的订单明细表格。

## 📋 功能特点

- **自动数据清理**：删除"BRL"字符，统一数值格式
- **智能筛选**：仅保留"已送达"订单，剔除无效数据
- **日期格式化**：统一转换为 YYYY/MM/DD 格式
- **订单编号去重**：重复订单自动添加后缀（-1, -2等）
- **自动计算**：商品单价 = 订单金额 ÷ 商品数量
- **完整字段填充**：包含物流信息、电商平台等全部字段

## 🚀 快速开始

### 方法一：直接打开HTML文件

1. 下载 `index.html` 和 `app.js` 文件到同一目录
2. 双击打开 `index.html` 文件
3. 上传B文件（OrderSKUList.xlsx）
4. 点击"开始处理"按钮
5. 处理完成后下载生成的订单明细文件

### 方法二：部署到GitHub Pages

#### 步骤1：创建GitHub仓库

1. 登录 GitHub
2. 创建新仓库（例如：`order-processor`）
3. 选择"Public"公开仓库

#### 步骤2：上传文件

将以下文件上传到仓库根目录：
- `index.html`
- `app.js`

可以通过以下方式上传：
- 网页界面直接拖拽上传
- 使用 Git 命令：
  ```bash
  git clone <your-repo-url>
  cd order-processor
  # 复制 index.html 和 app.js 到此目录
  git add .
  git commit -m "Initial commit"
  git push origin main
  ```

#### 步骤3：启用GitHub Pages

1. 进入仓库页面
2. 点击 "Settings" 标签
3. 左侧菜单找到 "Pages"
4. 在 "Source" 下拉框中选择 "main" 分支
5. 点击 "Save"

等待几分钟，GitHub Pages会自动部署，你会看到一个类似这样的URL：
```
https://<your-username>.github.io/order-processor/
```

#### 步骤4：访问应用

在浏览器中打开上述URL即可使用应用。

## 📊 使用说明

### 输入文件要求

- **文件格式**：Excel (.xlsx 或 .xls)
- **文件内容**：B文件（OrderSKUList），包含订单原始数据
- **必需列**：
  - Order ID (A列)
  - Order Substatus (D列)
  - Created Time (AC列)
  - Product Name (I列)
  - Quantity (L列)
  - Order Amount (X列)
  - Tracking ID (AK列)
  - Shipping Provider Name (AO列)

### 处理流程

1. **数据清理**：删除货币符号，统一数值格式
2. **数据筛选**：保留"已送达"订单，剔除金额为0的记录
3. **日期转换**：将美式日期格式转换为标准格式
4. **订单去重**：为重复订单编号添加唯一后缀
5. **模板填充**：按照A文件模板格式生成输出
6. **自动计算**：计算商品单价

### 输出文件

- **文件名**：订单明细.xlsx
- **工作表**：订单明细
- **列数**：11列（订单编号、订单日期、订单币种、订单金额、商品名称、商品数量、商品单价、店铺网址、快递单号、物流企业名称、电商平台英文名称）

## 🔧 技术栈

- **前端框架**：原生JavaScript（无框架依赖）
- **Excel处理**：SheetJS (xlsx库)
- **样式**：纯CSS，响应式设计
- **CDN依赖**：SheetJS通过CDN加载

## 📝 注意事项

1. **浏览器兼容性**：建议使用Chrome、Firefox、Edge等现代浏览器
2. **文件大小**：建议处理不超过10MB的Excel文件
3. **数据安全**：所有数据处理在浏览器本地完成，不会上传到服务器
4. **网络要求**：首次加载需要从CDN获取SheetJS库，需要网络连接

## 🐛 常见问题

**Q: 为什么处理后数据行数减少了？**
A: 系统会自动筛选掉非"已送达"状态的订单和金额为0的记录。

**Q: 订单编号为什么会变化？**
A: 如果原文件中存在重复的订单编号，系统会自动添加后缀（如10001-1, 10001-2）以确保唯一性。

**Q: 可以修改输出模板吗？**
A: 当前版本使用固定的A文件模板格式。如需自定义，可以修改 `app.js` 中的 `generateOutputFile` 函数。
