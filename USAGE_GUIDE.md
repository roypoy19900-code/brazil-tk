# 📖 使用指南

## 🎯 核心功能

本工具将B文件（OrderSKUList）自动处理并转换为标准化的订单明细表格。

### 处理步骤（自动化）

1. ✅ **数据清理**
   - 删除"BRL"货币符号
   - 将逗号替换为点号（统一数值格式）

2. ✅ **智能筛选**
   - 仅保留 Order Substatus = "已送达" 的订单
   - 剔除 Order Amount = 0 的记录
   - 删除第2行说明行

3. ✅ **日期格式化**
   - 输入：`07/21/2026 1:04:34 AM`
   - 输出：`2026/07/21`

4. ✅ **订单编号去重**
   - 重复订单自动添加后缀
   - 示例：`10001` → `10001`, `10001-1`, `10001-2`

5. ✅ **模板填充**
   - 订单编号 → A列
   - 订单日期 → B列
   - 订单币种 → C列（固定：BRL）
   - 订单金额 → D列
   - 商品名称 → E列
   - 商品数量 → F列
   - 商品单价 → G列（自动计算：金额÷数量）
   - 店铺网址 → H列（留空）
   - 快递单号 → I列
   - 物流企业名称 → J列
   - 电商平台 → K列（固定：TK）

## 🚀 三步完成

### 第1步：上传文件
- 点击上传区域或拖拽B文件（.xlsx）

### 第2步：开始处理
- 点击"开始处理"按钮
- 等待进度条完成（通常几秒内）

### 第3步：下载结果
- 查看处理统计
- 点击"下载订单明细文件"

## 💡 注意事项

- **浏览器要求**：Chrome、Firefox、Edge等现代浏览器
- **文件大小**：建议不超过10MB
- **数据安全**：所有处理在本地完成，不上传服务器
- **网络需求**：首次加载需要网络（获取SheetJS库）

## 🔍 输入文件要求

B文件应包含以下列（按列字母）：
- A列：Order ID
- D列：Order Substatus
- AC列：Created Time
- I列：Product Name
- L列：Quantity
- X列：Order Amount
- AK列：Tracking ID
- AO列：Shipping Provider Name

## ❓ 常见问题

**Q: 为什么数据行数减少了？**
A: 系统自动过滤了非"已送达"和金额为0的订单。

**Q: 订单编号为什么变了？**
A: 重复订单会自动添加后缀确保唯一性。

**Q: 可以批量处理吗？**
A: 当前版本一次处理一个文件，可以多次上传不同文件。

**Q: 支持哪些Excel格式？**
A: .xlsx 和 .xls 格式都支持。
